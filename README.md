# 同一ID斜め直線ステアリング課題 実験システム

斜め直線ステアリング課題をブラウザ上で実施するための小さな実験アプリです。開始時に本実験または予備実験と実験条件C1からC4を選択し，選択した条件をC番号順に提示します。本実験では反復試行，事後試行，アンケートまで進み，予備実験では選択条件を300試行ずつ実施します。実験完了時にはCSVが自動保存されます。

`same_id_diagonal_steering_experiment.html` をブラウザで開くと，そのまま実験を開始できます。ビルド手順や外部ライブラリは不要です。

## ファイル構成

| ファイル | 役割 |
| --- | --- |
| `same_id_diagonal_steering_experiment.html` | 実験アプリ本体です。画面，スタイル，キャンバス描画，試行制御，ログ表示，CSV/JSON保存を行います。 |
| `README.md` | 実験システムの仕様と使い方を説明します。 |
| `same_id_diagonal_steering_experiment.md` | 参加者向け・実験者向けの実験説明Markdownです。 |
| `steering_experiment_config.js` | 実験条件，本実験/予備実験の試行数，事後試行数，休憩間隔，エラー判定閾値などを定義します。 |

保存されるCSVとJSONは，次のファイル名になります。実験完了時と終了ボタン押下時はCSVが自動保存されます。

- `same_id_steering_{mode}_{participantId}_{conditionId}_{YYYYMMDD_HHMMSS}.csv`
- `same_id_steering_{mode}_{participantId}_{conditionId}_{YYYYMMDD_HHMMSS}.json`

## 実験目的

本実験では，ステアリングタスクにおける経路長 `A`，経路幅 `W`，および反復試行が，操作時間 `MT`，エラー，習熟過程に与える影響を分析します。

特に，同一の難易度指標 `ID = A / W` を持つ条件であっても，`A` と `W` の構成が異なる場合に，反復試行に伴う操作時間やエラーの変化が異なるかを検証します。

## 実験全体の流れ

1. 実験種別の選択と実験前説明
   - 本実験または予備実験をトグルで選択します。
   - 実験条件C1からC4をトグルで1つ以上選択します。
   - 所要時間は5分から10分程度です。
   - 文章によって，課題内容，操作方法，エラー条件，休憩方法を説明します。
2. 本実験: 反復試行
   - 選択した条件をC番号順に提示します。
   - 試行数は1条件あたり300試行です。
   - 100試行ごとに1分間の強制休憩を行います。
   - 課題は斜め直線ステアリングタスクです。
3. 本実験: 事後試行
   - 試行数は100試行です。
   - 反復試行と同様に，斜め直線ステアリングタスクを行います。
   - 反復試行で使用した条件に加えて，追加条件を提示できる設計です。
4. 本実験: アンケート回答
   - 所要時間は5分程度です。
   - 操作感，疲労感，課題の難しさなどを回答してもらいます。
5. 予備実験
   - 反復・事後・アンケートへの遷移は行いません。
   - 選択した条件をC番号順に1条件あたり300試行実施します。
   - 100試行ごとに一旦停止し，参加者が自分のタイミングで再開します。

## 実験対象者

- 日常的にマウスを使用している人を対象とします。
- 参加者情報として，以下を収集します。
- 参加者ID
- 年齢
- 利き手
- 日常的なマウス使用経験

## 実験内容

- 画面上に斜め直線の通路が表示されます。
- 参加者は，START領域からEND領域まで，通路に沿ってポインタを移動します。
- START領域とEND領域は，経路方向に描画可能範囲の端まで延長されています。
- 判定上も，START領域は開始点より手前，END領域は終了点より先の同幅領域として長さ上限なしで扱います。
- START領域内でポインタを押した時点から，移動時間 `MT` の計測を開始します。
- END領域内でポインタを離した時点で試行完了になります。
- 経路からはみ出した場合も，その試行は継続します。
- 経路逸脱の有無，逸脱回数，逸脱時間，最大はみ出し量はログと保存データに記録されます。
- 条件は `steering_experiment_config.js` の `conditions` で管理します。

## 反復試行条件

本実験の反復試行では，以下の4条件から1つ以上を選択します。複数選択した場合は，C1からC4の順に，1条件あたり300試行ずつ提示します。予備実験でも同じ条件選択を使い，1条件あたり300試行ずつ提示します。

| 条件ID | 条件名 | A | W | ID |
| --- | --- | ---: | ---: | ---: |
| C1 | 幅広条件 | 1000 | 50 | 20 |
| C2 | 幅狭条件 | 1000 | 20 | 50 |
| C3 | 距離大条件 | 1500 | 30 | 50 |
| C4 | 距離小条件 | 600 | 30 | 20 |

### 条件比較の意図

| 比較 | 目的 |
| --- | --- |
| C1 幅広条件 vs C2 幅狭条件 | 経路長 `A` を固定し，経路幅 `W` の違いが習熟過程に与える影響を調べる |
| C3 距離大条件 vs C4 距離小条件 | 経路幅 `W` を固定し，経路長 `A` の違いが習熟過程に与える影響を調べる |
| C1 幅広条件 vs C4 距離小条件 | 同一ID条件において，`A` と `W` の構成差が操作時間の変化に与える影響を調べる |
| C2 幅狭条件 vs C3 距離大条件 | 同一ID条件において，`A` と `W` の構成差がエラーや習熟速度に与える影響を調べる |

## エラーとして扱う条件

本実験では，操作上の問題を以下の2種類に分けます。

### 試行としてカウントし，分析時に処理するもの

以下の場合は，1試行としてカウントします。ただし，保存データにはエラー情報を記録し，分析時に除外または別指標として扱います。

- 経路からはみ出した場合
- 経路から大きくはみ出した場合
- ゴール到達直前，経路全体の約90%付近でポインタを離した場合

記録する主な情報は以下です。

- 経路逸脱の有無
- 経路逸脱の回数
- 経路外にいた合計時間
- 最大はみ出し量
- エラー種別
- エラー発生時の座標
- エラー発生時刻

### 試行としてカウントせず，やり直すもの

以下の場合は，その試行を有効試行としてカウントせず，同じ試行をやり直します。

- START領域から正しくスタートできていない場合
- 経路途中でポインタを離した場合
- ブラウザ外への移動やクリックミスなど，明らかな操作ミスが発生した場合
- その他，実験課題の遂行とは無関係な入力異常が発生した場合

## 事後試行

本実験では，反復試行後に100試行の事後試行を行います。

事後試行では，`steering_experiment_config.js` の `postConditions` に従い，標準ではC1からC4を25試行ずつ提示します。提示順は参加者IDをシードにした固定シャッフルです。

## 操作方法

1. `same_id_diagonal_steering_experiment.html` をブラウザで開きます。
2. 本実験または予備実験を選択します。
3. 実験条件C1からC4を1つ以上選択します。
4. 参加者IDを入力します。
5. 年齢，利き手，マウス使用経験を入力します。
6. 実験開始ボタンを押します。
7. 実験専用ページへ切り替わります。
8. ページ上部に，現在の種別，フェーズ，条件，試行数，中止，再開，休憩，終了ボタンが表示されます。
9. START領域からEND領域まで，表示された斜め直線の通路に沿って操作します。
10. END領域内でポインタを離すと，試行が記録されます。
11. 本実験では100試行ごとに1分間の強制休憩画面が表示されます。
12. 本実験では選択した全反復条件が終了すると，遷移画面を挟んで事後試行へ進みます。
13. 本実験では事後試行が終了すると，遷移画面を挟んでアンケートへ進みます。
14. 予備実験では選択条件ごとに300試行を行い，100試行ごとに手動再開の停止画面が表示されます。
15. 実験完了時にはCSVが自動保存され，開始画面に戻ります。途中で終了ボタンを押した場合も，記録済みの試行があればCSVを保存して開始画面に戻ります。

## キーボード操作

- `Escape`: 実行中の試行を中断します。
- `Enter`: 休憩中に次の試行へ進みます。

## `steering_experiment_config.js`

`window.SteeringExperimentConfig` が実験全体の設定オブジェクトです。

- `experimentModes`: 本実験と予備実験の試行数，事後試行，アンケート，強制休憩の有無を定義します。
- `practiceTrials`: 本実験の反復試行数を定義します。標準では300試行です。
- `postTrials`: 本実験の事後試行数を定義します。標準では100試行です。
- `pilotTrials`: 予備実験の試行数を定義します。標準では100試行です。
- `breakInterval`: 強制休憩を入れる試行間隔を定義します。標準では100試行ごとです。
- `forcedBreakSeconds`: 本実験の強制休憩秒数を定義します。標準では60秒です。
- `conditions`: 反復試行および予備試行で使用する条件ID，条件名，経路長 `amplitude`，経路幅 `width`，試行数 `trials` を定義します。
- `postConditions`: 事後試行で使用する条件を定義します。
- `steeringId`: 各条件の `amplitude / width` から自動計算されます。
- `errorCriteria.deviationThresholdPx`: はみ出しすぎを判定する閾値を定義します。
- `errorCriteria.nearGoalProgressThreshold`: ゴール到達直前でポインタを離したかを判定する進行率を定義します。標準では0.9です。

条件設定の例は以下です。

```js
{
  id: "C1",
  name: "幅広条件",
  label: "A固定・W広い",
  amplitude: 1000,
  width: 50,
  trials: 300
}
```

`steeringId` は `amplitude / width` から自動計算されます。試行数を変更したい場合は，`trials` または `practiceTrials` を変更してください。

## 条件を変更する方法

条件を変える場合は，`steering_experiment_config.js` の `conditions` を編集します。

```js
{
  id: "C1",
  name: "幅広条件",
  label: "A固定・W広い",
  width: 50,
  amplitude: 1000,
  trials: 300
}
```

事後試行の条件を変更する場合は，`postConditions` を編集します。事後試行全体の試行数を変更したい場合は，`postTrials` または `experimentModes.main.postTrials` を変更してください。

## 画面表示に関する設定

右側の実験キャンバスは，HTML内のCSSで表示比率を固定しています。条件リストやログが長くなった場合は，左側のパネルだけがスクロールし，右側の実験キャンバスの表示比率は変わりません。

斜め直線経路の角度，余白，最小表示幅を変更したい場合は，`steering_experiment_config.js` の `display` を編集します。

## 分析時の基本方針

分析では，以下の指標を条件ごと，試行ブロックごとに集計します。

- 平均移動時間 `MT`
- 中央値移動時間 `MT`
- エラー率
- 経路逸脱率
- 逸脱回数
- 最大はみ出し量
- 100試行ごとの変化
- 反復試行前半と後半の差
- 事後試行における条件間差

また，習熟過程の分析では，試行回数に伴う `MT` の低下をモデル化し，条件間で習熟速度や到達水準を比較します。
